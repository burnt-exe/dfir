DFIR301 – Module 4: Detect & Respond

A condensed, exam-ready cheat sheet summarizing slides 1–40 for fast revision and cybersecurity mastery. Use this guide to prepare for quizzes, classwork, and real-world DFIR operations.

1. Detect (DE) Function Overview

The Detect function helps identify cybersecurity events quickly to minimize damage and reduce attacker dwell time.

  • Identifies anomalies, threats, and IoCs.
  • Supports rapid response and containment.
  • Requires continuous monitoring & log analysis.
  • Improves threat visibility across networks & endpoints.

Core components:

  • Continuous monitoring
  • Event correlation
  • Threat intelligence
  • Adverse event analysis

2. Continuous Monitoring (DE.CM)

Monitoring includes networks, users, cloud, physical environments, and all digital assets.

Monitored elements:

  • Email & web traffic
  • Login events & authentication anomalies
  • File integrity changes
  • Endpoint behaviour
  • Cloud audit logs
  • Malicious domain/IP communications

Tuning is essential: reduces false positives & false negatives to maintain operational efficiency.

3. Detection Tools

SIEM

Aggregates logs, correlates events, alerts analysts.

Examples:

  • Splunk
  • ELK Stack
  • Microsoft Sentinel

EDR (Endpoint Detection & Response)

Monitors endpoint telemetry, identifies malicious behaviour, and supports incident replay.

Examples:

  • CrowdStrike Falcon
  • Microsoft Defender ATP
  • SentinelOne

IDS vs IPS

IDS Detects anomalies & alerts only
IPS Detects & blocks malicious activity

Threat Intelligence (CTI)

Improves detection accuracy by providing real-world indicators of attacker tooling, infrastructure, and behaviour.

4. Incident Triage & Analysis

Triage classifies incidents based on severity, volatility of data, and relevance to the organization.

Criteria:

  • Severity of threat
  • Type of impacted asset
  • Type of data involved
  • Attack vector & scope

OODA Loop (Decision Cycle):

  • Observe
  • Orient
  • Decide
  • Act

5. Evidence Handling & Chain of Custody

Proper handling ensures digital evidence is legally admissible and reliable.

Physical Evidence Examples:

  • Laptops, phones, PCs
  • Routers, switches, IoT devices
  • USB drives, SSD/HDD media
  • Smartwatches, cameras

Chain of Custody Requirements:

  • Unique evidence ID
  • Date/time of collection
  • Collector & handler names
  • Storage locations
  • Reason for each transfer

6. Quiz Preparation Checklist

  • Understand detection categories (DE.AE, DE.CM, DE.DP)
  • Know tools: SIEM, EDR, IDS, IPS, SOAR
  • Know differences: false-positive vs false-negative
  • Know the escalation chain
  • Know evidence-handling steps
  • Know ICS/CCI considerations

Mastering these concepts ensures readiness for both the quiz and real SOC/DFIR operations.
